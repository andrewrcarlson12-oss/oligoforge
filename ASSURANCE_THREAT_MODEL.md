# OligoForge Assurance threat model

## Protected assets

User-supplied assay sequences, sequence snapshots, metadata, experimental records, local paths, credentials and evidence-package integrity.

## Implemented controls

- Offline-first processing; no Assurance command fetches a URL or uploads a sequence.
- Explicit file arguments in the CLI; no path is inferred from sequence content.
- Compressed size, decompressed size, record-count, per-record length and total-base limits.
- FASTA alphabet validation and record-level rejection reasons.
- Exact-sequence deduplication without discarding the raw disposition ledger.
- Deterministic content hashes for AssaySBOMs, snapshots, deltas, scans, OFVRs and evidence packages.
- HTML escaping for human-readable reports.
- No executable spreadsheet formulas are generated by Validation Studio CSV exports.
- No claims of SOC 2, ISO 27001, HIPAA, GxP, FDA or IVDR compliance.

## Residual risks

- Local users can intentionally provide sensitive data; filesystem access control remains the deployer's responsibility.
- In-memory/web integration does not provide authenticated multi-tenant portfolio storage.
- Metadata may contain sensitive free text and should be reviewed before sharing an evidence package.
- Content hashes detect changes but are not digital signatures and do not establish author identity.
- ZIP/project-package import is not part of this slice; archive traversal defenses must remain in any future importer.
